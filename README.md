# 드림투어 행사 안내 페이지

행사 당일 참가자에게 링크로 공유하는 모바일 중심 안내 페이지입니다.
Node.js나 별도 빌드 과정 없이 동작하는 순수 HTML/CSS/JS 정적 사이트라서,
파일만 있으면 바로 웹 호스팅(GitHub Pages, Vercel 등)에 올릴 수 있습니다.

---

## 🔥 다음 행사 준비할 때 (운영팀용 요약)

콘텐츠를 바꾸는 방법은 두 가지입니다. **편한 쪽으로 아무거나 쓰시면 됩니다.**

### 방법 A. 편집기 화면에서 수정 (추천 — 코드 몰라도 OK)

1. **[editor.html](editor.html)** 파일을 더블클릭해서 브라우저로 엽니다.
   (또는 로컬 서버로 프로젝트 폴더를 열고 `editor.html`에 접속)
2. 화면의 폼에 행사명, 날짜, 일정, 연락처 등을 입력/수정합니다.
   - 스태프·일정·장소·체크리스트·FAQ는 "+ 추가" 버튼으로 항목을 늘리고,
     각 카드의 ↑ ↓ 버튼으로 순서를 바꾸고, "삭제" 버튼으로 지울 수 있습니다.
   - 대표 사진 / 스태프 사진은 "🖼️ 사진 선택" 버튼으로 내 컴퓨터의
     사진 파일을 직접 골라 넣을 수 있습니다 (미리보기가 바로 보입니다).
     "행사 기본 정보"의 **"📁 images 폴더 연결하기"** 버튼을 한 번 눌러 프로젝트의
     `images` 폴더를 연결해두면, 그다음부터 사진을 고를 때마다 자동으로 그 폴더에
     저장됩니다. (Chrome/Edge에서 로컬 서버로 열었을 때만 지원 — 지원하지 않는
     브라우저거나 `editor.html`을 더블클릭해서 열었다면, 고른 사진을 직접
     `images` 폴더에 복사해주세요. 파일은 이미 내 컴퓨터에 있으니 복사만 하면 됩니다)
3. 맨 아래 **"⬇ content.js 파일 저장"** 버튼을 누르면 새 `content.js` 파일이
   다운로드됩니다.
4. 다운로드된 파일을 프로젝트의 **js/content.js 위치에 덮어쓰기**한 뒤,
   평소처럼 GitHub에 올리면(push) 실제 사이트에 반영됩니다.

> ⚠️ editor.html은 참가자에게 공유하는 페이지가 아니라 운영팀만 쓰는 도구입니다.
> 서버 없이 브라우저 안에서만 동작하며(값이 자동 저장되지 않음), 페이지를
> 새로고침/닫기 전에 꼭 "content.js 파일 저장" 버튼을 눌러야 합니다.
> 사이트를 배포하면 `editor.html`도 함께 올라가지만, 홈/하단 메뉴 어디에서도
> 링크되어 있지 않고 비밀번호로 보호되지도 않습니다. 주소를 아는 사람은
> 누구나 열어볼 수 있으니, 민감한 정보(개인정보 등)는 입력하지 마세요.

### 방법 B. js/content.js 파일을 직접 텍스트 편집기로 수정

> **[js/content.js](js/content.js)**

이 파일 안에 행사명, 날짜, 공지사항, 담당자 연락처, 일정표, 지도 링크, FAQ,
설문조사 링크 등 화면에 보이는 거의 모든 텍스트가 들어있습니다.
파일을 열어보면 각 항목 위에 한글 설명 주석이 달려있어서, 무엇을 바꾸면
되는지 바로 알 수 있습니다. (메모장, VS Code 등 아무 텍스트 편집기로 열면 됩니다)

두 방법 모두 사진은 **[images](images)** 폴더에 넣고, `content.js`에서 지정한
파일명과 동일하게 저장하면 됩니다. (자세한 파일명은 [images/README.txt](images/README.txt) 참고)

### 항목별로 어디를 고치면 되는지 (방법 B 기준)

| 바꾸고 싶은 내용 | 수정할 위치 |
| --- | --- |
| 행사명, 날짜, 대표 사진, 한 줄 소개 | `content.js` 상단 "1. 행사 기본 정보" |
| 이번 회차에 필요 없는 섹션 통째로 끄기 (우천안내/스태프/집결정보/체크리스트/FAQ/설문) | `content.js` "1-1. 섹션 사용 여부" `sectionsEnabled` (또는 편집기에서 각 패널의 "이 섹션 사용" 체크박스) |
| 공지사항 배너 문구 (끄고 싶으면 `active: false`) | `content.js` "2. 공지사항 배너" |
| 우천 시 대응 안내 문구 | `content.js` "3. 우천 시 대응 안내" |
| 당일 담당자 이름/전화번호 (헤더·플로팅 버튼) | `content.js` "4. 당일 담당자 연락처" |
| 현장 스태프 소개 카드 | `content.js` "4. 당일 담당자 연락처" 아래 `staff` 배열 |
| 집결 시간/장소/지도 링크 | `content.js` "5. 집결 정보" |
| 시간별 일정표 (타임테이블), 각 일정 관련 사진 | `content.js` "6. 시간별 일정" `schedule` 배열 (각 항목의 `photo` 필드) |
| 오시는 길 페이지의 장소 목록/동선 순서 | `content.js` "7. 장소 목록" `locations` 배열 |
| 준비물 체크리스트 항목 | `content.js` "8. 준비물 체크리스트" |
| 자주 묻는 질문 | `content.js` "9. FAQ" |
| 설문조사 링크 / 예상 소요시간 | `content.js` "10. 설문조사 링크" |
| 사진 파일 | `images/` 폴더 |
| 전체 디자인/색상 (파란색 브랜드 컬러 등) | `css/style.css` 상단 `:root` 변수 (개발자용) |

**HTML(`index.html` 등)이나 `js/main.js`, `js/home.js` 같은 화면 구성 코드는
평소에는 건드릴 필요가 없습니다.** 구조 자체를 바꾸고 싶을 때만 손대면 됩니다.

## 🖥️ 로컬에서 미리보기

빌드 과정이 필요 없습니다. `index.html` 파일을 더블클릭해서 브라우저로 열면

모바일 화면으로 보고 싶다면, 브라우저 개발자 도구(F12)의 기기 툴바를
켜서 확인하거나, 실제 스마트폰으로 파일을 전송해서 열어보세요.

## 🚀 배포 방법

### 방법 1. Vercel (추천 — 링크 발급이 가장 간단)

1. [vercel.com](https://vercel.com) 가입 후 이 프로젝트 폴더를 GitHub 저장소로 올립니다.
2. Vercel에서 "New Project" → 해당 저장소 선택 → Framework Preset은
   **Other**로 두고 그대로 배포(Deploy)합니다. (빌드 명령 없이 정적 파일 그대로 배포됩니다)
3. 배포가 끝나면 `https://프로젝트명.vercel.app` 같은 링크가 생성됩니다.
4. 다음 행사 때는 `content.js`와 사진만 교체하고 다시 GitHub에 푸시하면
   Vercel이 자동으로 재배포합니다.

### 방법 2. GitHub Pages (무료, 별도 서비스 가입 불필요)

1. 이 폴더를 GitHub 저장소에 푸시합니다.
2. 저장소의 **Settings → Pages** 메뉴에서 Source를 `main` 브랜치, 폴더는
   `/ (root)`로 설정합니다.
3. 잠시 후 `https://아이디.github.io/저장소명/` 링크로 접속할 수 있습니다.

## 📸 참가자 사진 업로드 기능 설정하기 (선택, 최초 1회만)

일정 중 특정 활동(예: 팀별 미션, 단체 활동)에서 **참가자가 직접 사진을 올리고,
운영팀은 편집기에서 그 사진들을 모아볼 수 있는 기능**입니다. 이 사이트는 서버가
없는 정적 사이트라서, 사진을 실제로 저장해둘 공간(무료 클라우드 저장소인
[Firebase](https://firebase.google.com) Storage)이 필요합니다. **행사마다 하는
설정이 아니라, 딱 한 번만 해두면 계속 재사용됩니다.**

설정 전에는 이 기능이 자동으로 꺼진 상태로 동작하므로(에러 없이 "준비 중" 문구만
표시됨), 급하지 않다면 건너뛰어도 사이트의 다른 기능에는 전혀 영향이 없습니다.

### 1) Firebase 프로젝트 만들기

1. [Firebase 콘솔](https://console.firebase.google.com)에 구글 계정으로 접속합니다.
2. **"프로젝트 추가"** → 이름 입력(예: `dreamtour-photos`) → Google Analytics는
   꺼도 무방 → **"프로젝트 만들기"**.

### 2) 사진 저장 공간(Storage) 켜기

1. 왼쪽 메뉴 **빌드 → Storage** 선택 → **"시작하기"**.
2. 보안 규칙은 일단 기본값으로 두고, 위치는 `asia-northeast3 (서울)`을 추천합니다.

### 3) 이 사이트와 연결할 설정값(config) 복사하기

1. 왼쪽 위 톱니바퀴 아이콘 → **"프로젝트 설정"** → **"일반"** 탭 아래로 스크롤.
2. **"내 앱"** 에서 `</>` (웹 앱) 아이콘 클릭 → 앱 닉네임 입력(아무 이름) → **"앱 등록"**.
3. 화면에 나오는 `firebaseConfig` 값들을 **[js/firebase-config.js](js/firebase-config.js)**
   파일의 각 항목에 그대로 붙여넣습니다. (`YOUR_API_KEY_HERE` 같은 자리에)

### 4) 업로드 용량/형식 제한 설정하기 (보안 규칙)

Storage 메뉴의 **"Rules"** 탭에서 아래 내용으로 바꾸고 **"게시"**를 누르세요.
(참가자는 사진을 올릴 수만 있고, 다른 사람 파일을 지우거나 바꿀 수는 없게 하며,
10MB 이하 이미지 파일만 허용하는 규칙입니다)

```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /uploads/{allPaths=**} {
      allow read: if true;
      allow write: if request.resource.size < 10 * 1024 * 1024
                   && request.resource.contentType.matches('image/.*');
    }
  }
}
```

### 5) 특정 일정에 업로드 기능 켜기

- **편집기(editor.html)** 에서 "⑦ 시간별 일정" 패널 → 해당 일정 카드의
  **"📸 참가자에게 이 활동 사진 업로드 받기"** 체크박스를 켜세요.
- 또는 `content.js`에서 해당 일정 항목의 `photoUpload.enabled`를 `true`로 바꿔도 됩니다.
- 체크박스를 켠 상태에서 같은 카드 안의 **"🔄 업로드된 사진 확인"** 버튼을 누르면
  참가자가 올린 사진들을 그 자리에서 바로 볼 수 있습니다.

### 참고

- Firebase 무료 요금제(Spark)는 저장 용량 5GB, 하루 다운로드 1GB까지 무료입니다.
  보통 규모의 워크숍이라면 충분하지만, 여러 회차를 계속 쌓아서 쓴다면 가끔
  Firebase 콘솔에서 사용량을 확인해주세요.
- `js/firebase-config.js` 안의 값들은 "비밀번호"가 아니라 브라우저에 공개적으로
  노출되는 일반적인 설정값입니다(구글의 정상적인 사용 방식). 실제 보안은 위 4)의
  Storage 규칙으로 지킵니다.
